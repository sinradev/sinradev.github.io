## Vue d'ensemble

Les opérations en masse permettent de modifier plusieurs issues en une seule action depuis la vue liste. Sélectionnez deux issues ou plus pour activer le panneau d'actions groupées.

## Sélectionner des Issues

Dans la vue liste, cochez la case à gauche de chaque issue à modifier. Cochez la case d'en-tête pour tout sélectionner sur la page courante.

Le nombre d'issues sélectionnées s'affiche dans le panneau d'actions en bas de page.

## Actions Disponibles

| Action | Description |
|--------|-------------|
| **Définir le statut** | Met à jour le statut de développement pour toutes les issues sélectionnées |
| **Assigner** | Réassigne toutes les sélectionnées à un utilisateur ou une équipe |
| **Définir la release** | Déplace les issues sélectionnées vers une autre release |
| **Définir le cycle** | Réassigne les sélectionnées à un autre cycle |
| **Ajouter un label** | Ajoute un label à toutes les issues sélectionnées |
| **Définir la plateforme** | Met à jour la plateforme pour toutes les sélectionnées |

## Cas d'Usage Courants

**Planification de sprint** : filtrez les issues d'une release, sélectionnez celles qui entrent dans le cycle suivant, assignez-les en une action.

**Réassignation d'équipe** : lors d'un changement d'organisation, réassignez rapidement toutes les issues d'un développeur partant vers un autre.

**Nettoyage du backlog** : sélectionnez les issues sans release et affectez-les en masse à la prochaine release planifiée.

**Mise à jour de statut** : en fin de sprint, marquez toutes les issues terminées en un clic.

## Points d'Attention

Les opérations en masse sont irréversibles en une seule action. Vérifiez votre sélection avant d'appliquer. Pour les grandes sélections, utilisez les filtres pour isoler précisément les issues concernées avant de sélectionner.
