## Ce que Fait le Backlog

Le backlog est la surface de planification où se prennent les décisions de périmètre. Vous voyez toutes les capabilities non assignées d'un côté et vos releases planifiées de l'autre. Faites glisser une capability dans une release pour l'inclure dans le périmètre de cette version.

C'est là que vous répondez à : "qu'est-ce qui va dans cette release vs la prochaine ?"

## Glisser-Déposer

Les capabilities sont des cartes déplaçables dans le backlog. Déposez une capability sur une release pour l'assigner. La release recalcule immédiatement sa charge de capacité.

Pour retirer une capability d'une release, faites-la glisser vers le pool non assigné ou vers une autre release.

## Avertissements de Capacité

Chaque carte de release affiche son utilisation actuelle de la capacité. Quand le temps total estimé des capabilities assignées dépasse la capacité cible de la release, Sinra affiche un avertissement de capacité.

C'est le moment de prendre une vraie décision :
- Retirer une capability de cette release et la repousser à la suivante
- Augmenter la capacité de la release (ajouter plus de cycles ou prolonger le délai)
- Accepter la surcharge et planifier des heures supplémentaires

L'avertissement ne vous bloque pas. Il rend le compromis visible pour que la décision soit explicite.

## Capabilities Non Assignées

Les capabilities non assignées à aucune release apparaissent dans le pool non assigné. Ce sont des fonctionnalités planifiées qui n'ont pas encore été programmées. Garder le pool non assigné propre accélère les revues de planification.

## Relation avec les Cycles

Assigner une capability à une release définit le travail (quoi). Assigner ses issues à des cycles planifie le travail (quand). Ces deux décisions sont séparées et peuvent se produire dans n'importe quel ordre.
