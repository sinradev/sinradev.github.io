## ¿Qué son las Páginas?

Las páginas son documentos en texto enriquecido dentro de Sinra. Sirven como wiki interno: definiciones de negocio, reglas funcionales, decisiones de arquitectura, guías de incorporación y vocabulario compartido.

A diferencia de herramientas externas como Notion o Confluence, las páginas viven en el mismo espacio de trabajo que tus issues y capabilities. No hay cambio de contexto para encontrar una definición o una regla de negocio.

## Escribir una Página

El editor de páginas soporta:
- Encabezados y estructura
- Negrita, cursiva, código
- Listas con viñetas y numeradas
- Imágenes
- `@mentions` de usuarios y otras páginas

El texto enriquecido hace que las páginas sean útiles para todo, desde una definición corta hasta un documento de especificación completo.

## Mencionar Páginas en Issues

Cualquier cuerpo de issue o capability puede hacer `@mention` de una página. La mención se convierte en un enlace clicable. Cuando el lector hace clic, ve la versión actual de la página, no una copia que podría estar desactualizada.

Esto crea un sistema de referencia vivo: el producto escribe reglas de negocio en Páginas, los ingenieros referencian esas páginas en specs e issues, y el enlace siempre está actualizado.

## Categorías y Labels

Las páginas soportan categorías y labels para la organización. Usa categorías para separar áreas (Producto, Ingeniería, Proceso) y labels para etiquetado más fino.

## Compartir

Las páginas se comparten en toda la organización por defecto. Cualquier miembro puede leer y editar páginas. Los permisos siguen la configuración de roles de la organización.
