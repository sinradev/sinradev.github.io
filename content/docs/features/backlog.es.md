## Qué Hace el Backlog

El backlog es la superficie de planificación donde ocurren las decisiones de alcance. Ves todas las capabilities sin asignar por un lado y tus releases planificadas por el otro. Arrastra una capability a una release para incluirla en el alcance de esa versión.

Aquí es donde respondes a: "¿qué va en esta release vs en la siguiente?"

## Arrastrar y Soltar

Las capabilities son tarjetas arrastrables en el backlog. Suelta una capability sobre una release para asignarla. La release recalcula inmediatamente su carga de capacidad.

Para quitar una capability de una release, arrástrala de vuelta al pool sin asignar o a una release diferente.

## Advertencias de Capacidad

Cada tarjeta de release muestra su utilización actual de capacidad. Cuando el tiempo total estimado de las capabilities asignadas supera la capacidad objetivo de la release, Sinra muestra una advertencia de capacidad.

Este es el momento de tomar una decisión real:
- Quitar una capability de esta release y moverla a la siguiente
- Aumentar la capacidad de la release (añadir más ciclos o ampliar el plazo)
- Aceptar la sobrecarga y planificar horas extra

La advertencia no te bloquea. Hace visible el compromiso para que la decisión sea explícita.

## Capabilities Sin Asignar

Las capabilities no asignadas a ninguna release aparecen en el pool sin asignar. Son funcionalidades planificadas que todavía no se han programado. Mantener el pool sin asignar ordenado hace más rápidas las revisiones de planificación.

## Relación con los Ciclos

Asignar una capability a una release define el trabajo (qué). Asignar sus issues a ciclos programa el trabajo (cuándo). Estas dos decisiones son separadas y pueden ocurrir en cualquier orden.
