## Test Overview

This template documents user feedback on icon recognition and semantic accuracy. Each user story has different testing requirements.

**General Approach**:
- Show icon(s) to participant without text labels (except where noted)
- Ask open-ended question about what the icon represents
- Record verbatim response
- Rate response as correct/acceptable/incorrect
- Calculate accuracy percentage

---

## User Story 1: Feature Icons

**Icons Tested**: editor, progress, collaboration, release, multiapp, workflows, carryover (7 total)

**Testing Method**:
1. Show each icon at 96px (standard desktop size)
2. No text labels visible
3. Ask: "What feature or capability does this icon represent?"
4. Record response
5. Rate: Correct / Acceptable / Incorrect

**Success Criteria**: 75%+ of participants give correct/acceptable answers

## User Story 3: Methodology Icons

**Icons Tested**: agile, waterfall (2 total)

**Participant Profile**: 8+ users familiar with project management methodologies

**Testing Method**:
1. Show agile and waterfall icons side-by-side
2. Ask: "What methodology does each icon represent?"
3. Ask: "How do they visually differ? Why is the difference important?"
4. Record responses
5. Rate understanding: Correct / Partial / Incorrect

**Success Criteria**: 70%+ of participants understand visual distinction and difference between methodologies
